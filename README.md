# Сириус.Аренда

**Конкурс:** Всероссийский ИТ-раунд  
**Трек:** Разработка и управление программным обеспечением  
**Проект:** сервис бронирования пространств

## Описание проекта

В рамках цифрового развития федеральной территории «Сириус» нужен сервис, который позволит студентам, преподавателям и гостям Университета бронировать переговорные комнаты и учебные пространства.

В проекте реализован **бэкенд-компонент** — REST API для управления пространствами, бронированиями и расписанием. Дополнительно сделан веб-интерфейс для работы с API.

### Цель проекта

Разработать серверное приложение, предоставляющее API для:

- управления переговорными комнатами и пространствами;
- бронирования пространства на конкретное время;
- проверки доступности пространства;
- просмотра расписания.

## Технологии

| Что | Выбор |
|-----|-------|
| Язык | Python 3 |
| Фреймворк | **FastAPI** (рекомендуемый вариант из задания) |

## Инструкция по запуску

### 1. Скачать проект

```bash
git clone https://github.com/drefeks695-creator/sirius-rent.git
cd sirius-rent
```

Если репозиторий уже скачан — перейдите в его папку в терминале.

### 2. Установить зависимости (один раз)

**Windows (PowerShell или cmd):**

```bash
python -m venv venv
venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

**Linux / macOS:**

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

> Если `pip install` падает с ошибкой сети, попробуйте зеркало PyPI:
> `pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple --trusted-host pypi.tuna.tsinghua.edu.cn`

### 3. Запустить сервер

**Windows:** дважды кликните **`start.bat`** (запускает сайт на локалке).

Или из терминала:

```bash
venv\Scripts\activate
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Дождитесь строки `Uvicorn running on http://127.0.0.1:8000` и откройте в браузере: **http://127.0.0.1:8000/ui/**

### Частые проблемы

| Проблема | Решение |
|----------|---------|
| `start.bat`: Virtual environment not found | Сначала выполните шаг 2 (создание `venv` и `pip install`) |
| `Permission denied` при `python -m venv venv` | В терминале есть `(venv)` — выполните `deactivate`, удалите папку `venv`, повторите шаг 2 |
| Ошибка скачивания пакетов с pypi.org | Установите зависимости через зеркало (см. шаг 2) |
| Порт 8000 занят | Закройте другой сервер и снова запустите `start.bat` |
| Сайт открыт, но API не работает | Открывайте только **http://127.0.0.1:8000/ui/**, не файл `static/index.html` напрямую |

### Docker Compose

## Функциональные требования (MVP)

### 2.1. Управление пространствами

CRUD для переговорных и аудиторий. Каждое пространство содержит:

- уникальный идентификатор (`id`);
- название (`name`);
- вместимость (`capacity`);
- список оборудования (`equipment`: проектор, доска, конференц-связь и т.д.).

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/rooms` | Создание пространства (admin) |
| GET | `/rooms` | Список всех пространств |
| GET | `/rooms/{id}` | Детальная информация |
| PUT | `/rooms/{id}` | Редактирование (admin) |
| DELETE | `/rooms/{id}` | Удаление (admin) |

**Фильтрация:**  
`GET /rooms?capacity_min=8&capacity_max=50&equipment=проектор`

**Тело запроса POST /rooms:**

```json
{
  "name": "Переговорная А-101",
  "capacity": 32,
  "equipment": ["проектор", "доска", "конференц-связь"]
}
```

**Ответ (пример):**

```json
{
  "id": 1,
  "name": "Переговорная А-101",
  "capacity": 32,
  "description": "...",
  "image_url": "/ui/images/room-1.png",
  "equipment": ["проектор", "доска", "конференц-связь"],
  "bookings_blocked": false,
  "open_time": "08:00",
  "close_time": "22:00"
}
```

### 2.2. Бронирование

Бронирование на конкретную дату и время. Каждое бронирование содержит:

- уникальный идентификатор (`id`);
- идентификатор пространства (`room_id`);
- дату и время начала (`start_time`);
- дату и время окончания (`end_time`);
- имя пользователя (`user_name` — кто забронировал);
- статус (`status`: `active` / `cancelled` / `completed`).

**Проверка занятости:** при пересечении с другой активной бронью сервер возвращает ошибку **409 Conflict**.

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/bookings` | Создание бронирования (JWT, с проверкой доступности) |
| DELETE | `/bookings/{id}` | Отмена бронирования |
| GET | `/rooms/{id}/bookings?date=YYYY-MM-DD` | Брони комнаты на дату |

**Тело POST /bookings:**

```json
{
  "room_id": 1,
  "start_time": "2026-07-15T10:00:00",
  "end_time": "2026-07-15T11:00:00"
}
```

**Заголовок:** `Authorization: Bearer <token>`

**Ответ (пример):**

```json
{
  "id": 1,
  "code": "SR-20260715-A1B2",
  "room_id": 1,
  "user_name": "student",
  "start_time": "2026-07-15T10:00:00",
  "end_time": "2026-07-15T11:00:00",
  "status": "active"
}
```

### 2.3. Хранение данных

Используется **реляционная БД SQLite** через SQLAlchemy.

Основные таблицы: `users`, `rooms`, `bookings`.  
Схема спроектирована с внешними ключами и проверкой конфликтов на уровне приложения.

### 2.4. Обработка ошибок

API возвращает понятные сообщения с HTTP-статусами:

| Код | Когда |
|-----|-------|
| 200 OK | Успешный запрос |
| 201 Created | Объект создан |
| 400 Bad Request | Ошибка валидации (некорректные данные, время вне рабочих часов) |
| 401 Unauthorized | Нет или неверный JWT-токен |
| 403 Forbidden | Недостаточно прав |
| 404 Not Found | Объект не найден |
| 409 Conflict | Пространство уже занято на выбранное время |

## Дополнительные баллы (реализовано)

По заданию эти пункты опциональны — в проекте они сделаны:

1. **Аутентификация JWT** с ролями `user` и `admin`. Пользователь управляет только своими бронированиями.
   - `POST /auth/register` — регистрация
   - `POST /auth/login` — получение токена
2. **Поиск свободных пространств:**
   - `GET /rooms/available?start=...&end=...&capacity=...`
3. **Юнит-тесты** — более 50 тестов (требование: не менее трёх):
   ```bash
   pytest -v
   ```
4. **Docker Compose** — `Dockerfile`, `docker-compose.yml`.
5. **Веб-интерфейс** — страницы со списком комнат, формой бронирования, календарём, профилем, админкой (`static/`).

## Документация API

Полная интерактивная документация: **http://127.0.0.1:8000/docs**

### Аутентификация

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/auth/register` | Регистрация (`username`, `password`) |
| POST | `/auth/login` | Вход, ответ: `{ "access_token": "...", "token_type": "bearer" }` |

### Пространства и расписание

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/rooms/available` | Свободные комнаты на интервал |
| GET | `/rooms/{id}/availability` | Проверка доступности одной комнаты |
| GET | `/rooms/{id}/schedule` | Расписание на день |
| GET | `/rooms/{id}/schedule/month` | Календарь на месяц |

### Профиль

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/profile/me` | Профиль и список своих броней |
| PATCH | `/profile/me` | Обновление ФИО, телефона, почты |

### Примеры запросов

**Вход:**

```bash
curl -X POST http://127.0.0.1:8000/auth/login \
  -d "username=student&password=student123"
```

**Создание бронирования:**

```bash
curl -X POST http://127.0.0.1:8000/bookings \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d "{\"room_id\":1,\"start_time\":\"2026-07-15T10:00:00\",\"end_time\":\"2026-07-15T11:00:00\"}"
```

**Свободные комнаты:**

```bash
curl "http://127.0.0.1:8000/rooms/available?start=2026-07-15T10:00:00&end=2026-07-15T11:00:00&capacity=8"
```

**Брони комнаты на дату:**

```bash
curl "http://127.0.0.1:8000/rooms/1/bookings?date=2026-07-15"
```
